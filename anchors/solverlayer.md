# solverlayer.eth

Semantic anchor representing **solver terminology surfaces** in intent-based transaction workflows.

## Context

"Solvers" generally refer to actors that compete to satisfy user intents by constructing optimal transaction bundles.
This concept is often discussed in the context of intents, orderflow auctions, and execution markets.

In many discussions, solver terminology is more app-layer / protocol-adjacent than strictly L1-canonical.
This anchor exists to track the term surface without claiming L1 enshrinement.

## Registry Role

- Track terminology stabilization for solvers across intent discussions
- Distinguish L1-relevant usage from L2/app-layer execution markets
- Prevent naming drift (solver vs filler vs searcher) by citing primary usage where available
