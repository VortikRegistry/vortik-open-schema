# Security Hourglass — Coordination Compression Model

This document presents a structural model describing how coordination pressure compresses across emerging Ethereum primitives.

It focuses on how key layers interact:

- Order Flow & Execution Markets  
- Builder Markets & ePBS  
- Inclusion Enforcement (FOCIL)  
- Commitment & Preconfirmation Layers  
- Finality (SSF research track)  

---

## Why this matters

Ethereum is transitioning from isolated components to a coordinated execution pipeline.

As this happens:

→ Terminology stabilizes  
→ Coordination surfaces compress  
→ Naming becomes infrastructure  

This model explores that transition.

---

## Positioning

This is not an official specification.

It is an independent research artifact aligned with:

- EIP-7732 (ePBS)  
- EIP-7805 (FOCIL / Inclusion Lists)  
- Preconfirmation research  
- SSF research
