# Vortik Registry v0.6.5 public release notes

## Purpose

These notes describe a public documentation, trust, and source-state update for the Vortik Registry. They are not a formal protocol release, do not update registry state, and do not change the package or registry version.

## Public documentation updates

This documentation release records the public-facing state of the repository around:

- README and documentation index improvements that make the source-of-truth model easier to follow;
- trust and validation documentation for the public-safety scanner, registry validation, integrity checks, and derived-output sync discipline;
- the research notes index as the navigation point for public interpretive notes;
- the glossary as explanatory material that does not create registry entries;
- the candidate backlog as monitoring material, not a promotion queue;
- BAL source notes for Block-Level Access Lists, `block_access_list_hash`, eth/71 BAL exchange, and snap/2 BAL-based state healing;
- FOCIL / `inclusionlist.eth` source-state clarification, including EIP-7805 as the primary EIP source and EIP-7773's Declined for Inclusion state for Glamsterdam;
- ePBS / `epbs.eth` source-state clarification, including EIP-7732 as the primary ePBS specification source and EIP-7773's Scheduled for Inclusion state for Glamsterdam;
- public safety scanner and validation discipline for keeping public registry-facing material technical, source-grounded, and free of prohibited nontechnical strategy.

## Registry-state non-changes

This release note confirms:

- no registry entries were added;
- no ENS anchors were added;
- no registry status, classification, or type was changed;
- no schema `const` values were changed;
- no prohibited nontechnical strategy is included;
- no generated outputs, maps, index files, market files, scripts, package files, `registry.json`, or `registry.schema.json` are changed by this note.

## How to read this note

This note summarizes documentation and audit work only. Source-of-truth registry state remains in `registry.json` and the versioned schema files. Human-facing generated or derived docs remain derived from source-of-truth files and do not override `registry.json`, schemas, `sources.md`, or anchor documents.
