# Contributing to Vortik Semantic Registry

Thank you for your interest in improving the Vortik Semantic Registry.

This repository tracks terminology stabilization and semantic anchors across emerging Ethereum coordination primitives.

The registry is a **documentation and research tool**, not a governance process.

It does not define protocol standards.

---

# Contribution Principles

Vortik operates as a **research-first registry**.

Contributions should:

- reference **primary sources** whenever possible
- avoid speculative or marketing terminology
- focus on **terminology stabilization**
- preserve minimal schema design
- avoid protocol advocacy or design proposals

The goal of the registry is **semantic clarity**, not protocol governance.

---

# Reporting Terminology Drift

If you observe terminology divergence across research discussions,
please open an Issue including:

- source links
- observed terminology
- suggested canonical term (if appropriate)

This helps the registry track how terminology evolves across Ethereum research.

---

# Proposing New Anchors

When proposing a new anchor, please include:

- clear description of the coordination primitive surface
- at least **two primary research references**
- evidence that the terminology appears across multiple discussions

Anchors should represent **coordination primitives**, not products, companies, or applications.

---

# Proposing Schema Updates

Schemas represent **machine-readable semantic metadata**.

Schema updates should:

- remain minimal
- preserve compatibility with existing schema structure
- include references to primary sources where relevant

Avoid introducing speculative fields or protocol assumptions.

---

# Updating Sources

Sources should prioritize **primary technical material**, including:

- Ethereum Research forum
- EIPs
- core developer discussions
- primary technical blog posts

Avoid secondary commentary or speculative analysis when possible.

---

# Editing Existing Anchors

Anchor documents should remain **short orientation documents**.

When editing anchors:

- avoid expanding them into full specifications
- preserve conceptual neutrality
- ensure terminology reflects primary sources

Detailed metadata should remain in the corresponding schema.

---

# Pull Request Guidelines

Before submitting a pull request:

- confirm that references are valid
- ensure terminology is consistent with existing anchors
- avoid speculative claims about protocol adoption

When possible, include links to relevant research discussions.

---

# Scope Reminder

Vortik documents **semantic surfaces**, not implementations.

The registry exists to track conceptual convergence across Ethereum coordination layers and research discussions.
